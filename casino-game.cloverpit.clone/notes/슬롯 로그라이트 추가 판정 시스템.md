아래는 개발자에게 전달 가능한 형태로 정리한
“추가 판정(Chain Evaluation) 시스템” 기획 의뢰서입니다.

# 슬롯 로그라이트 추가 판정 시스템 기획 의뢰서

## 목적

현재 슬롯 시스템은:

```text id="4y3nq5"
라인 보상 계산
→ 코인 지급
```

수준으로 동작하고 있음.

이를:

```text id="3c0w4o"
“연쇄적으로 판정이 폭주하는 로그라이트 슬롯”
```

구조로 확장하고자 함.

핵심 목표는:

```text id="8lpnxy"
한 번의 스핀 결과가
여러 단계의 추가 판정을 연쇄적으로 발생시키는 것
```

이다.

---

# 1. 핵심 개념

## 기존 슬롯

```text id="ohw4vc"
라인 판정 1회
→ 종료
```

---

## 목표 구조

```text id="b6rx1y"
라인 판정
→ 심볼 개수 판정
→ 멀티라인 판정
→ 패턴 판정
→ 아이템 판정
→ 콤보 판정
→ 이벤트 판정
```

순으로 추가 계산이 이어져야 함.

---

# 2. 시스템 목표 경험

플레이어가:

```text id="w3xqjo"
“계속 뭔가 추가로 터진다”
```

라고 느끼도록 설계.

즉:

```text id="yrj0yv"
결과 확인
→ 추가 보상 발생
→ 또 추가 효과 발생
→ 콤보 상승
→ 추가 이벤트 발생
```

구조를 의도.

---

# 3. 추가 판정 처리 순서 (매우 중요)

아래 순서대로 고정 처리.

---

# STEP 1 — 라인 판정

## 역할

기본 슬롯 보상 계산.

---

## 예시

```text id="84npqf"
체리 3개
벨 2개
```

---

## 처리

```text id="mjlwm7"
라인별 기본 보상 계산
```

---

# STEP 2 — 심볼 개수 판정

## 역할

전체 릴에 등장한 심볼 개수 기준 추가 보상.

---

## 예시

```text id="km9yk3"
클로버 총 4개
```

---

## 처리 예시

| 조건          | 효과    |
| ----------- | ----- |
| 클로버 3개 이상   | 추가 코인 |
| 다이아 2개 이상   | 배수 증가 |
| Skull 3개 이상 | 디버프   |

---

## 중요

라인과 무관하게:

```text id="jlwm77"
“전체 화면 기준”
```

으로 계산.

---

# STEP 3 — 멀티라인 판정

## 역할

동시 다중 라인 성공 보너스.

---

## 예시

```text id="nndh9g"
라인 3개 동시 성공
```

---

## 처리

```text id="qzwxv0"
MULTI WIN BONUS
```

추가 지급.

---

## 추천 규칙

| 라인 수   | 효과       |
| ------ | -------- |
| 2라인    | Combo +2 |
| 3라인    | Combo +3 |
| 4라인 이상 | 추가 배수    |

---

# STEP 4 — 패턴 판정

## 역할

특수 배열 형태 판정.

---

## 예시

| 패턴       | 효과     |
| -------- | ------ |
| 좌우 대칭    | 추가 배수  |
| 같은 심볼 연속 | 연쇄 보너스 |
| X 형태     | 특수 이벤트 |
| 전체 동일 색상 | 색상 보너스 |

---

## 목적

단순 줄 계산이 아니라:

```text id="mjlwm8"
“배열 자체를 읽는 느낌”
```

을 주기 위함.

---

# STEP 5 — 아이템 판정

## 역할

획득한 강화 아이템 연쇄 발동.

---

## 예시

### Lucky Clover

```text id="jlwm88"
클로버 포함 라인당 +50%
```

---

### Golden Bell

```text id="jlwm89"
벨 보상 ×2
```

---

## 중요

아이템은:

```text id="jlwm90"
다른 판정 결과를 기반으로 연쇄 발동
```

해야 함.

즉:

```text id="jlwm91"
라인 판정 결과
→ 아이템 발동
→ 추가 계산
```

구조.

---

# STEP 6 — 콤보 판정

## 역할

이번 스핀 결과 기반 콤보 증가.

---

## 규칙

| 상황    | Combo |
| ----- | ----- |
| 일반 성공 | +1    |
| 큰 성공  | +2    |
| 멀티라인  | 추가 증가 |
| 잭팟    | +5    |
| 실패    | 초기화   |
| 역잭팟   | 초기화   |

---

## 중요

콤보는:

```text id="jlwm92"
마지막 단계에 계산
```

할 것.

---

# STEP 7 — 이벤트 판정

## 역할

특수 조건 기반 추가 이벤트.

---

## 예시

| 조건        | 이벤트             |
| --------- | --------------- |
| 777       | JACKPOT         |
| 666       | REVERSE JACKPOT |
| 5라인 이상    | OVERDRIVE       |
| Combo x20 | FEVER MODE      |

---

## 목표

플레이어에게:

```text id="jlwm93"
“이번 판이 폭주한다”
```

는 느낌 제공.

---

# 4. 핵심 정책

---

# 정책 1 — 판정은 순차적이어야 함

절대:

```text id="jlwm94"
모든 계산을 동시에 처리하지 말 것
```

---

## 이유

플레이어는:

```text id="jlwm95"
“연쇄적으로 터지는 느낌”
```

을 받아야 함.

---

# 정책 2 — 판정 로그를 출력할 것

예시:

```text id="jlwm96"
Cherry Line +120
Multi Line Bonus +200
Lucky Clover +50%
Combo x5
FINAL WIN +980
```

---

## 목적

플레이어가:

```text id="jlwm97"
왜 코인이 증가했는지
```

이해 가능해야 함.

---

# 정책 3 — 연출은 단계별로 발생

잘못된 방식:

```text id="jlwm98"
모든 이펙트 동시에 출력
```

---

## 목표 방식

```text id="jlwm99"
라인 점등
→ 추가 보너스
→ 아이템 발동
→ 콤보 증가
→ 최종 폭발
```

순으로 진행.

---

# 정책 4 — 플레이어가 계산 폭주를 느껴야 함

핵심 목표:

```text id="jlwm100"
“처음엔 작은 당첨이었는데
계속 추가 보상이 붙어서
엄청난 결과가 된다”
```

는 감각.

---

# 5. 개발 요구사항

---

# 필수 구현

```text id="jlwm101"
- 단계별 판정 시스템
- 판정 우선순위 시스템
- 연쇄 발동 구조
- 판정 로그 UI
- 단계별 연출 큐
- 콤보 연계 구조
```

---

# 권장 구조

```text id="jlwm102"
SpinResultEvaluator

 ├─ evaluateMultiLines()
 ├─ evaluatePatterns()
 ├─ evaluateItems()
 ├─ evaluateCombo()
 └─ evaluateEvents()
```

# 6. 최종 목표

이 시스템의 핵심은:

```text id="jlwm103"
“단순 슬롯”
```

이 아니라:

```text id="jlwm104"
“판정이 연쇄 폭주하는 로그라이트 슬롯”
```

을 만드는 것이다.
