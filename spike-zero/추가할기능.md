# 📄 관제실 대사 기능 요구사항 (최종 정리)

---

## 0. 배경

배경 현재 구현된 관제실 대사 기능을 고도화한다

## 1. 적용 이벤트

관제실 대사는 아래 3개 이벤트에서만 출력

1. 스테이지 시작
2. 보스 등장 전 (워닝 이후)
3. 보스 클리어 후

---

## 2. 보스 워닝 연출 변경 (핵심)

### 현재

* 워닝 메시지 출력과 동시에 보스 등장

### 변경

* 워닝 메시지 종료 후 보스 등장

```plaintext id="yvtfxc"
워닝 메시지 → 종료 → 관제실 대사 → 보스 등장
```

### 목적

* 보스 등장 전 브리핑 및 대사 연출 추가
* 전투 진입 전 흐름 강화

---

## 3. 보스 클리어 연출 흐름

```plaintext id="ffz4if"
보스 파괴 → 관제실 대사 → 스테이지 클리어 UI → 다음 스테이지
```

---

## 4. 대사 구성 규칙

* 이벤트당 최대 4줄
* 한 줄은 짧고 간결하게 구성
* 설명형 대사 지양
* 캐릭터 성격 반영 필수

---

## 5. 캐릭터 설정

### 관제사 (신입)

* 냉정, 단답형
* 감정 표현 거의 없음
* 정보 위주 발언

예:

* “고위험 개체 감지.”
* “패턴 변화 확인.”
* “…”

---

### 플레이어 (베테랑 조종사)

* 여유 있고 능글맞음
* 가벼운 농담 + 상황 인지
* 필요 시 진지한 반응

예:

* “어이 신입, 긴장했나?”
* “이건 좀 빡센데.”
* “왔네.”

---

## 6. 연출 규칙

* 대사 출력 시 게임 일시 정지
* 대사 종료 후 다음 단계 진행
* Test Stage 포함 동일하게 동작해야 함

---

## 7. 전체 흐름 요약

```plaintext id="q2cazn"
[스테이지 시작]
→ 대사 → 게임 시작

[보스 구간]
→ 워닝 → 종료 → 대사 → 보스 등장

[보스 클리어]
→ 대사 → 클리어 UI → 다음 스테이지
```

---

## 8. 핵심 변경 포인트

* 워닝 이후 보스 등장 구조로 변경
* 이벤트 3개로 제한
* 짧은 대사 기반 티키타카 연출
